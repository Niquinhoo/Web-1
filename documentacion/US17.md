# User Story #17 - Normalizacion de IDs

**Codigo:** `#main-s2-us17`

**Nombre:** Normalizacion de IDs

### Detalle

La historia pide normalizar y validar IDs de producto antes de usarlos para evitar errores silenciosos.

### Implementacion realizada

Se agrego la funcion:

- `normalizeId(rawId)` en `services/productsService.js`

Reglas aplicadas:

- ID no numerico => invalido
- ID numerico <= 0 => invalido
- ID valido => se normaliza a string numerico

Uso en rutas:

- `routes/productos.router.js` ahora valida `/:id` con `normalizeId`.
- Si el ID es invalido, responde `400` y renderiza `views/pages/400/400-page.ejs`.
- Si el ID es valido pero no existe en catalogo, responde `404`.

### Validacion

Cumplido:

- Existe `normalizeId()`.
- ID no numerico devuelve `400`.
- ID numerico inexistente devuelve `404`.
- No se usa API.
