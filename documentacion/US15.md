# User Story #15 - Servicio de Productos

**Codigo:** `#main-s2-us15`

**Nombre:** Servicio de productos

### Detalle

La historia pide encapsular la logica de lectura y filtrado de productos en un modulo de servicios para mantener controladores limpios.

### Implementacion realizada

Se creo:

- `services/productsService.js`

Este servicio concentra la logica de productos:

- listado completo
- sugeridos
- top pedidos
- busqueda por ID
- relacionados
- aleatorios
- filtrado por categoria
- fallback de imagen

Se refactorizo:

- `controllers/productController.js`

Ahora el controller funciona como capa delgada y delega en `productsService`.

### Validacion

Cumplido:

- Existe `services/productsService.js`.
- Los controladores usan el servicio.
- La logica de negocio de productos deja de vivir en el controller.
- No se usa API.
- No se usa DB.
