# User Story #13 - 500

**Codigo:** `#main-s2-us13`

**Nombre:** Pagina 500

### Detalle

La historia pide manejar errores internos con middleware de error y renderizar una vista dedicada `500.ejs`.

### Implementacion realizada

Se agrego manejo global de errores en `app.js`:

- middleware de error con firma `(error, req, res, next)`
- log del error en servidor
- respuesta `status(500)` con render de vista dedicada

Se creo la vista:

- `views/pages/500/500-page.ejs`

Comportamiento de la vista:

- muestra estado `500` de forma clara
- no expone stack trace ni detalles internos
- ofrece accion para volver al inicio

### Criterios de validacion

Cumplido:

- Se dispara middleware de error cuando llega un `next(error)` o excepcion de servidor.
- Se renderiza vista `500.ejs`.
- No se filtra informacion sensible al usuario final.
- No usa API.
