***Código:** #main-s1-us6*

***Nombre:** Construir página de producto*

En este caso me ayude más de la inteligencia artificial para setear fallbacks para que cuando el usuario ingrese a un url de un producto que no existe, lo redirija a una pagina de error sugiriendo nuevos productos existentes
Ademas hice una base de datos compartida hardcodeada que me va a servir para modularizar los productos y dar el salto a una arquitectura de monolito a 3 capas de manera más sencilla, teniendo que ajustar a funciones de flujo real y borrando el hardcodeo
Aca dejo un resumen de como lo hizo el agente la seccion de la base de datos y el fallback

1. Base de Datos Compartida (`data/db.js`)
Para poder acceder a los productos tanto desde la página de inicio (Home) como desde las rutas dinámicas de los productos, extrajimos los arreglos hardcodeados (`productos`, `categorias`, `publicidades`) que estaban en `routes/index.router.js` hacia un nuevo módulo de datos en `data/db.js`.
- Se agregó la propiedad `id` y `category` a cada producto para facilitar su búsqueda y la relación entre productos.
- Se agregó la propiedad `description` a los productos para nutrir la vista de detalles.

2. Enrutador Dinámico de Productos (`routes/productos.router.js`)
Se actualizó el router de productos para manejar rutas dinámicas mediante `GET /producto/:id`:
- **Flujo Exitoso:** Si el producto existe, se renderiza la vista `product-detail-page`. Además, se filtran productos de la misma categoría (excluyendo el actual) para mostrarlos en la sección de **Productos Relacionados**.
- **Flujo de Error (404):** Si el producto no existe (por ejemplo, `/producto/1234`), se captura el error y se renderiza la vista `product-not-found-page`. Para evitar que el usuario abandone el sitio, se envía un arreglo aleatorio de 4 productos para la sección de **Productos Sugeridos**.
