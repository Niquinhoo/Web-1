# User Story #10 - Listado por Categoria

**Codigo:** `#main-s2-us10`

**Nombre:** Listado de productos por categoria

### Detalle

La historia pedia habilitar una vista por categoria en `/categories/:category` para listar todos los productos coincidentes usando la fuente local, reutilizando la card existente y mostrando un estado vacio amigable.

### Implementacion final

La US10 esta **completamente implementada** y operativa en la navegacion principal.

- **Ruta:** `GET /categories/:category` (con alias `/category/:category`) habilitada en `routes/categories.router.js`.
- **Filtro:** Se resuelve en `services/productsService.js` mediante `getProductsByCategory(category)`, aplicando normalizacion de strings (tildes, espacios, mayusculas).
- **Controlador:** Gestiona el renderizado de la pagina de categoria enviando la lista filtrada de productos.
- **Vista:** `views/pages/product/category-products-page.ejs` reutiliza el organismo `product-grid` para mantener la consistencia visual.
- **Barra de Navegacion:** La barra de categorias (`category-bar`) vincula dinamicamente a cada ruta de categoria funcional.

### Validacion contra la story

Cumplido al 100%:

- [x] Endpoint `/categories/:category` funcional.
- [x] Filtro de productos por categoria desde fuente local.
- [x] Renderizado de productos coincidentes reutilizando cards.
- [x] Mensaje amigable (estado vacio) si no hay productos en la categoria elegida.
- [x] Vista muestra el nombre de la categoria actual.
- [x] No usa API/fetch/base de datos externa.

### Conclusion

La US10 cierra el flujo de navegacion del catalogo, permitiendo al usuario filtrar por interes de manera fluida y cumpliendo con todos los criterios de arquitectura (SSR + MVC) y de negocio solicitados.
