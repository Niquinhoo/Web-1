# User Story #11 - Sin Stock

**Codigo:** `#main-s2-us11`

**Nombre:** Indicador de producto sin stock

### Detalle

La historia pide que, cuando un producto tenga `stock === 0`, se indique visualmente como "Sin stock" y no se pueda agregar al carrito.

### Implementacion realizada

Se implemento en frontend y backend:

- En la card de producto (`views/partials/molecules/home/product-card.ejs`):
- se evalua `Number(product.stock) === 0`
- se muestra badge `Sin stock`
- el boton `Agregar` queda deshabilitado
- En el detalle de producto (`views/partials/molecules/product/product-info.ejs`):
- se muestra el mismo badge
- el boton `Agregar al Carrito` queda deshabilitado
- En el backend de carrito (`controllers/cartController.js`):
- `addProductToCart` valida stock
- si `stock` es numerico y `<= 0`, retorna `false` y no agrega
- En la ruta de alta al carrito (`routes/cart.router.js`):
- se respeta el resultado de `addProductToCart`
- si falla la validacion, no se agrega el item

### Criterios de validacion

Cumplido:

- Si `stock === 0`, se muestra indicador visual "Sin stock".
- El boton de agregar aparece deshabilitado.
- No se permite agregar productos sin stock (validacion de servidor).
- No se usa API ni base de datos.

### Notas tecnicas

- La validacion de stock es defensiva: solo considera "sin stock" cuando `stock` existe y es numerico menor o igual a cero.
- Si un producto no define `stock`, se mantiene el comportamiento actual (disponible).
